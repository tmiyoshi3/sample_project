# F-03-02: 製品詳細画面 - ユースケース一覧

## 業務領域: 製品カタログ

## UC-010: 製品の詳細情報を確認して調達・在庫・技術判断の材料を得る

### アクター
- 全ロール（ADMIN, MANAGER, BUYER, WAREHOUSE_STAFF, VIEWER）
- 主にBUYER（発注判断のための価格・サプライヤー・代替品確認）、WAREHOUSE_STAFF（在庫状況・物理情報・保管要件の確認）、MANAGER（カタログ管理・変更履歴の追跡）が利用

### 目的
製品一覧で見つけた製品の詳細情報（基本属性・価格・在庫・仕様・画像・代替品・サプライヤー・ドキュメント・変更履歴）をタブ切替で確認し、調達判断・在庫管理・技術評価などの業務に必要な情報を得る。必要に応じて関連ドキュメントをダウンロードしたり、代替品やサプライヤーの詳細画面に遷移して比較検討する。

### 事前条件
- ユーザーがシステムにログイン済みである（UC-001完了済み）
- 製品一覧画面（UC-008）または他画面のリンクから製品を選択している
- 対象製品がデータベースに存在する

### 基本フロー
1. ユーザーが製品一覧画面の行クリック（F-03-01-012）等から製品詳細画面（/products/:id）にアクセスする
2. 画面が初期表示され、以下のAPIが呼び出される:
   - GET /api/products/{id} → 製品詳細情報（基本情報・価格・仕様・画像・ドキュメント・代替品ID）
   - GET /api/products/{id}/change-log → 変更履歴
   - GET /api/products?page=0&size=5 → 代替品表示用（先頭5件、自身除外）
3. ヘッダ領域（F-03-02-001）に製品名・ステータスバッジ・SKU・カテゴリ・メーカーが常時表示される
4. 基本情報タブ（F-03-02-005、デフォルト選択）が表示され、以下のセクションを確認する:
   - 基本属性: 製品名・SKU・説明・カテゴリ・メーカー・ステータス
   - 価格・数量（F-03-02-006）: 単価・単位・最低発注数・リードタイム
   - 物理情報（F-03-02-007）: 重量・寸法
   - 在庫状況（F-03-02-008）: 合計在庫・予約済み・利用可能
   - 管理情報（F-03-02-009）: 登録日・最終更新日・備考
5. ユーザーが必要に応じてタブを切り替えて詳細情報を確認する:
   - 仕様タブ（F-03-02-010）: 製品スペック（キーバリューテーブル）
   - 画像タブ（F-03-02-011）: 製品画像（グリッド表示、メインバッジ付き）
   - 代替品タブ（F-03-02-012）: 代替製品一覧（SKU・製品名・メーカー・単価・ステータス）
   - サプライヤータブ（F-03-02-013）: 取扱サプライヤー一覧（サプライヤー名・SKU・単価・リードタイム・優先バッジ）
   - ドキュメントタブ（F-03-02-014）: 関連ドキュメント（仕様書・カタログ等、ダウンロード可能）
   - 変更履歴タブ（F-03-02-015）: 変更タイムライン（日時・フィールド・旧値→新値・変更者）
6. ユーザーが情報確認を完了し、「← 一覧に戻る」ボタン（F-03-02-002）で製品一覧画面に戻る

### 代替フロー
- A1: 代替品の詳細を確認する場合 → 代替品タブの「詳細」ボタンをクリックし、代替製品の詳細画面（/products/:alternativeId）に遷移する
- A2: サプライヤーの詳細を確認する場合 → サプライヤータブの「詳細」ボタンをクリックし、サプライヤー詳細画面（/suppliers/:supplierId）に遷移する
- A3: ドキュメントをダウンロードする場合 → ドキュメントタブの「ダウンロード」ボタンをクリックし、PDFファイル等をダウンロードする
- A4: 製品情報を編集する必要がある場合 → 「編集」ボタン（F-03-02-003）をクリックし、製品編集画面（/products/:id/edit、F-03-05）に遷移する
- A5: 基本情報タブのみで情報確認が完了する場合 → ヘッダと基本情報タブの情報で十分であれば、他タブを開かずに一覧に戻る

### 例外フロー
- E1: 製品が存在しない場合（404） → エラーバナー「製品情報の取得に失敗しました。」が表示される。「一覧に戻る」ボタンで製品一覧に戻る
- E2: 変更履歴の取得に失敗した場合 → 変更履歴タブが空リスト表示。コンソールにエラー出力
- E3: 代替品データの取得に失敗した場合 → 代替品タブが空リスト表示。コンソールにエラー出力
- E4: セッション期限切れ → AuthInterceptorがBearerトークンの自動更新を試行。失敗時はKeycloakログインにリダイレクト
- E5: ドキュメントのファイルパスが空の場合 → ダウンロード操作が実行されない（静かに失敗）

### 事後条件
- ユーザーが製品の詳細情報（価格・在庫・仕様・サプライヤー・変更履歴等）を把握している
- 必要に応じてドキュメントをダウンロード済みである
- 調達判断・在庫管理・技術評価等の次のアクションに進める状態になっている

## UC-011: 不要になった製品をカタログから除外する

### アクター
- 全ロール（ADMIN, MANAGER, BUYER, WAREHOUSE_STAFF, VIEWER）
- 本来はADMIN, MANAGERのみが操作すべきだが、@RolesAllowed未設定のため全ロールで操作可能（技術的負債）

### 目的
廃番・取扱終了などの理由で不要になった製品を、カタログの有効な製品リストから除外する。物理削除ではなく論理削除（ステータスをDISCONTINUEDに変更）により、履歴やトレーサビリティを維持しつつカタログから除外する。

### 事前条件
- ユーザーがシステムにログイン済みである（UC-001完了済み）
- 製品詳細画面（/products/:id）を表示している（UC-010の途中状態）
- 対象製品のステータスがDISCONTINUED以外である

### 基本フロー
1. ユーザーが製品詳細画面（/products/:id）で対象製品の情報を確認する
2. ユーザーが「削除」ボタン（F-03-02-004、btn-danger）をクリックする
3. 確認ダイアログ「製品の削除」が表示される:
   - メッセージ: 「この製品を削除してもよろしいですか？この操作は取り消せません。」
   - ボタン: 「キャンセル」「削除する」
4. ユーザーが「削除する」ボタンをクリックする
5. DELETE /api/products/{id} が呼び出される
6. バックエンドでproductService.changeStatus(id, "DISCONTINUED")が実行される:
   - ステータス遷移バリデーション
   - ProductChangeLogにステータス変更履歴を記録
   - 監査ログに記録
7. 204 No Content レスポンスを受信する
8. 製品一覧画面（/products）に自動遷移する

### 代替フロー
- A1: 削除をキャンセルする場合 → 確認ダイアログの「キャンセル」ボタンをクリックし、ダイアログが閉じて製品詳細画面に戻る。データ変更なし

### 例外フロー
- E1: 既にDISCONTINUEDの製品を削除しようとした場合 → 422 Unprocessable Entity（ValidationException）。エラーメッセージ「製品の削除に失敗しました。」が表示される
- E2: 製品が存在しない場合 → 404 Not Found（EntityNotFoundException）。エラーメッセージが表示される
- E3: API通信エラー → エラーメッセージ「製品の削除に失敗しました。」が表示される。画面遷移しない
- E4: セッション期限切れ → AuthInterceptorがBearerトークンの自動更新を試行。失敗時はKeycloakログインにリダイレクト

### 事後条件
- 製品のステータスがDISCONTINUEDに変更されている
- ProductChangeLogにステータス変更履歴が記録されている
- 監査ログに削除操作が記録されている
- ユーザーが製品一覧画面に遷移している
- 製品一覧画面で当該製品のステータスが「廃番」（DISCONTINUED）と表示される

### 関連機能ID
- F-03-02-004（削除ボタン）

### 関連API
- DELETE /api/products/{id}（製品論理削除: ステータスをDISCONTINUEDに変更）

### 根拠
- 画面操作: Chrome DevTools MCPで削除ボタンクリック → 確認ダイアログ表示 → キャンセルでダイアログ閉じを確認（実際の削除操作は未実行）
- スクリーンショット: screenshots/F-03/F-03-02-004_delete_confirm.png（確認ダイアログ）
- ソースコード: product-detail.component.ts（confirmDelete, deleteProduct, cancelDelete: 確認ダイアログ表示→DELETE API呼び出し→/products遷移）
- ソースコード: ProductResource.java:334-341（deleteProduct: changeStatus(id, "DISCONTINUED")による論理削除）
- ソースコード: ProductServiceBean.java:421-466（changeStatus: ステータス遷移バリデーション、ProductChangeLog記録）

### confidence
High

### 未確認事項
- 在庫がある製品の削除時の挙動（在庫制約チェックの有無。在庫がある状態でDISCONTINUEDにできるか）
- 既にDISCONTINUEDの製品を再度削除した場合の具体的なエラーメッセージ内容
- 削除後の製品一覧画面での当該製品の表示確認（DISCONTINUED製品が一覧に残るか、ステータスフィルタで非表示になるか）
- ロール別の削除ボタン表示制御（本来VIEWER/BUYERには削除を制限すべき可能性あり）
