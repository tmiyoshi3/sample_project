# Output Migration Step 001: テスト結果テーブルを集計サマリーに変換

## 背景

skill 10（現行E2E）とskill 11（移行後実装）がワークログにシナリオ単位の全件テスト結果テーブルを記載していた。この情報はskill 12のregression-report/F-XX.mdにも記載されるため、同じデータが最大3箇所に重複していた。シナリオ数が多い機能ではワークログだけで数十行を消費していた。

## 修正方針

ワークログのテスト結果は**集計値のみ**とし、シナリオ単位のテーブルは削除する。詳細はregression-reportへの参照で代替する。

## 対象ファイル

`spec/worklog/F-XX.md` の中で、以下のようなセクションが該当する:
- 「テスト結果」セクション（skill 10の実行記録）
- 「移行後E2Eテスト結果」セクション（skill 11の実行記録）

## 修正手順

各 `spec/worklog/F-XX.md` を開き、以下のパターンを探す。

### skill 10の記録

**修正前:**
```markdown
### テスト結果

| シナリオ | 結果 | 備考 |
|---|---|---|
| 正常ログイン | PASS | |
| パスワード誤り | PASS | |
| 未入力送信 | PASS | |
| 権限なしアクセス | FAIL | 仕様と現行画面に差分あり |
...（シナリオ数分の行が続く）
```

**修正後:**
```markdown
### テスト結果サマリー

- 対象シナリオ数: X
- PASS: X / FAIL: X / SKIP: X
- FAIL シナリオ: （FAILがある場合のみシナリオ名と理由を簡潔に記載）

※シナリオ別の詳細は regression-report/F-XX.md を参照
```

### skill 11の記録

**修正前:**
```markdown
### 移行後E2Eテスト結果

| シナリオ | 現行結果 | 移行後結果 | 差分 | 分類 |
|---|---|---|---|---|
| 正常ログイン | PASS | PASS | なし | - |
| パスワード誤り | PASS | PASS | なし | - |
| 権限なし | PASS | FAIL | エラーメッセージ文言差異 | 意図的変更 |
...（シナリオ数分の行が続く）
```

**修正後:**
```markdown
### 移行後E2Eテスト結果サマリー

- 対象シナリオ数: X
- 現行: X pass / X fail
- 移行後: X pass / X fail
- 差分: X件（意図的変更: X、移植漏れの可能性: X ...）

※シナリオ別の詳細比較は regression-report/F-XX.md を参照
```

## 注意事項

- 既存のテーブルから pass/fail の件数を集計してサマリーに変換すること
- FAILや差分があるシナリオは、シナリオ名と概要を1行で残してよい（テーブルにはしない）
- regression-report/F-XX.md がまだ作成されていない機能（skill 12未実施）は、テスト結果テーブルをそのまま残してよい。skill 12実施時に新フォーマットで記録される

## 完了報告

修正完了後、以下の形式で報告すること:

### 修正したファイル一覧
ファイルごとに、修正箇所数を記載する。

例:
```
- spec/worklog/F-01.md — テスト結果2箇所をサマリー化
- spec/worklog/F-02.md — テスト結果1箇所をサマリー化（skill 12未実施のため移行後テスト結果はそのまま）
```

### 未修正のファイル
修正対象だが未修正のファイルがあれば、ファイル名とその理由を記載する。

- [ ] 集計値が元のテーブルの件数と一致すること
- [ ] regression-report未作成の機能ではテーブルがそのまま残っていること
