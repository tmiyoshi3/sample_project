# Output Migration Step 004: 差分ゼロのregression-reportを簡潔化

## 背景

`spec/regression-report/F-XX.md` のテンプレートが、差分の有無に関わらず常にフルフォーマット（比較概要テーブル、差分一覧、意図的変更の理由、結論）で記載されていた。差分がゼロの場合でも全シナリオを列挙した比較テーブルが作成され、シナリオ40個超の機能では差分ゼロなのに100行以上のレポートになっていた。

## 修正方針

- 差分ゼロのレポートは**簡潔フォーマット**に変換する（集計サマリー + 結論のみ、最大10行程度）
- 差分ありのレポートは、**差分があるシナリオのみ**を差分一覧に残し、差分なしのシナリオ行を削除する

## 対象ファイル

`spec/regression-report/F-XX.md`（作成済みの全ファイル）

## 修正手順

### 差分ゼロのレポートの場合

差分一覧が全て「差分なし」であるレポートを探す。

**修正前:**
```markdown
# リグレッション比較レポート

## 対象機能ID: F-01-01-001

### 比較概要

| 項目 | 現行 | 移行後 | 結果 |
|---|---|---|---|
| E2Eテスト pass数 | 35/35 | 35/35 | |
| 画面表示差分 | - | - | 差分なし |
| 操作差分 | - | - | 差分なし |
| API差分 | - | - | 差分なし |

### 差分一覧

| # | 差分内容 | 現行の挙動 | 移行後の挙動 | 分類 | 対応方針 | 対応先skill |
|---|---|---|---|---|---|---|
（なし）

### 意図的変更の理由

（なし）

### 未確認事項

- なし

### 結論

- 差分総数: 0件
...
```

**修正後:**
```markdown
# リグレッション比較レポート: F-01-01-001

## サマリー
- 対象シナリオ数: 35
- 現行: 35 pass / 0 fail
- 移行後: 35 pass / 0 fail
- 差分: なし

## 結論
全シナリオで差分なし。移行後実装は現行仕様を正しく再現している。
```

### 差分ありのレポートの場合

**修正ポイント:**
1. 比較概要テーブルをサマリー形式（箇条書き）に変換する
2. 差分一覧から「差分なし」の行を全て削除し、差分があるシナリオのみ残す
3. 意図的変更の理由・未確認事項・結論はそのまま残す

**修正後の形式:**
```markdown
# リグレッション比較レポート: F-XX-YY-ZZZ

## サマリー
- 対象シナリオ数: X
- 現行: X pass / Y fail
- 移行後: X pass / Z fail
- 差分: N件

## 差分一覧

| # | シナリオ | 差分内容 | 現行の挙動 | 移行後の挙動 | 分類 | 対応方針 | 対応先skill |
|---|---|---|---|---|---|---|---|
| 1 | （差分のあるシナリオのみ記載） | ... | ... | ... | ... | ... | ... |

## 意図的変更の理由
（該当がある場合のみ記載）

## 未確認事項
...

## 結論
- 差分総数: N件
- 意図的変更: X件
- 移植漏れの可能性: X件
- 仕様不明: X件
- テスト不備: X件
- 現行バグの可能性: X件
- 未確認: X件
```

## 注意事項

- regression-report/F-XX.md が存在しない大分類（skill 12未実施）はスキップしてよい
- 差分ゼロかどうかは「結論」セクションの「差分総数: 0件」で判断できる
