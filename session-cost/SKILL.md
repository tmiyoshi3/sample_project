# Session Cost — Claude Codeセッションコスト集計

作業中のプロジェクトに紐づくClaude Codeセッションのトークン使用量とAPI費用概算を集計して表示する。

## 仕組み

Claude Codeは `~/.claude/projects/<project-dir>/` 以下にセッションごとの会話ログ（`.jsonl`）を保存している。各APIレスポンスに含まれる `message.usage` からトークン数を取得し、モデルごとの料金テーブルで費用を算出する。

## 使い方

### 0. 料金テーブルの確認・更新（スクリプト実行前に必ず行う）

スクリプト実行前に、料金テーブルが最新かどうかを確認する。

1. `<skill-dir>/scripts/pricing.json` を読み込み、`pricing_url` のURLをWebFetchでフェッチする
2. フェッチ結果に応じて以下のように対応する:

**正常にフェッチでき、料金情報が取得できた場合:**
- ページ上のAPI料金（入力・出力・キャッシュ書込・キャッシュ読取）と `pricing.json` の値を比較する
- 差異がある場合 → `models`（料金値）と `last_updated`（今日の日付）を更新
- ページに新しいモデルが掲載されていた場合 → `models` に追加
- URLが変わっていた（リダイレクト等）場合 → `pricing_url` も更新

**URLが無効（404、料金情報が見つからない等）の場合:**
- 「Anthropic API pricing」等のキーワードでWebSearchして正しいURLを特定する
- `pricing_url`・`models`・`last_updated` の3点をすべて更新する

**フェッチ失敗（オフライン等）の場合:**
- 既存の `pricing.json` をそのまま使用する
- スクリプト出力後に「オフラインのため料金確認ができませんでした（pricing.json の最終更新: YYYY-MM-DD）」と注記する

**重要: 料金を更新する際は、必ず `pricing_url`・`models`・`last_updated` の3点をセットで更新する。**

### 1. ユーザーの時間指定を解釈する

ユーザーは自然言語で時間を指定する。以下のように解釈してISO形式に変換する:

| ユーザーの指定 | 変換例 |
|---|---|
| 「今日の14時以降」 | `2026-06-03T14:00:00+09:00` |
| 「昨日から」 | `2026-06-02T00:00:00+09:00` |
| 「6月1日以降」 | `2026-06-01T00:00:00+09:00` |
| 「今週」 | 今週月曜日の00:00 |
| 指定なし | `--since` 引数を省略（全期間） |

ユーザーのタイムゾーンは、セッションのタイムスタンプやシステム情報から推定する。

### 2. スクリプトを実行する

```bash
python3 <skill-dir>/scripts/calculate_cost.py --cwd <CWD> --since <ISO-DATETIME>
```

- `--cwd`: 現在の作業ディレクトリ（プロジェクトパス）。省略時はカレントディレクトリ。
- `--since`: 集計開始時刻（ISO形式）。省略時は全期間。
- `--format`: `text`（デフォルト、Markdown表形式）または `json`（プログラム処理用）。

### 3. 結果をそのまま表示する

スクリプトの出力はMarkdown形式なので、そのまま表示すればよい。

出力に `⚠️ 警告: 未登録モデルを検出` が含まれていた場合は、ステップ0の手順に戻り、`pricing.json` に該当モデルの正しい料金を追加してからスクリプトを再実行する。

## 出力内容

- **セッション別サマリー**: セッションごとのAPI呼び出し回数とコスト
- **モデル別内訳**: モデルごとに入力・キャッシュ書込・キャッシュ読取・出力のトークン数とコスト
- **料金注記**: 使用したモデルの料金レートと注意事項
- **料金テーブル最終更新日**: いつ時点の料金で計算したか
